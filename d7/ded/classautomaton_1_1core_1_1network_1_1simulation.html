◆ handle_request()

void automaton::core::network::simulation::handle_request ( uint32_t  src,
uint32_t  dest 
)

This event is created when the other endpoint has called connect(). On_requested is called and if it returns true, new event with type accept is created, new connection is created from this endpoint to the other, the new connection's state is set to connected and on_connect in acceptor's handler is called. If the connections is refused, new event type refuse is created.
